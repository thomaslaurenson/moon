# Python Documentation

Standards and conventions for documenting Python library packages using Sphinx.

## When to Add Docs

Any installable library package should have a `docs/` directory and Sphinx setup. Scripts-only projects do not require Sphinx.

## Dependencies

Add a `docs` optional dep group in `pyproject.toml`:

```toml
[project.optional-dependencies]
docs = [
    "sphinx>=7.3.7",
    "furo>=2024.5.6",
    "sphinx-autodoc-typehints>=2.3.0",
    "myst-parser>=4.0.0",
    "sphinx-copybutton>=0.5.2",
]
```

Install with `uv sync --extra docs`.

## Directory Structure

```
docs/
  conf.py
  index.rst
  api/
    index.rst      # committed source file
    generated/    # auto-generated by autosummary - do not edit manually
  _build/
    html/         # gitignored
```

Git ignore rules:
- ignore `docs/_build/`
- ignore `docs/api/generated/`
- do not ignore `docs/api/` (you must commit `docs/api/index.rst`)

Avoid custom autosummary templates unless there is a very strong reason. They add hidden maintenance and package-specific coupling.

## conf.py

```python
from __future__ import annotations

import re
import sys
from datetime import datetime
from importlib.metadata import PackageNotFoundError, version as metadata_version
from pathlib import Path

PROJECT_NAME = "<package>"
BASE_DIR = Path(__file__).resolve().parents[1]
sys.path.insert(0, str(BASE_DIR))


def _version_from_pyproject() -> str:
  pyproject = (BASE_DIR / "pyproject.toml").read_text(encoding="utf-8")
  match = re.search(r'^version\s*=\s*"([^"]+)"\s*$', pyproject, re.MULTILINE)
  if not match:
    raise RuntimeError("Could not determine project version from pyproject.toml")
  return match.group(1)

project = PROJECT_NAME
author = "..."
year = datetime.now().year
copyright = f"{year}, {author}"

try:
    release = metadata_version(PROJECT_NAME)
except PackageNotFoundError:
  release = _version_from_pyproject()

version = release

extensions = [
    "sphinx.ext.autodoc",
    "sphinx.ext.autosummary",
    "sphinx.ext.napoleon",
    "sphinx.ext.viewcode",
    "sphinx.ext.intersphinx",
    "sphinx_autodoc_typehints",
    "sphinx_copybutton",
    "myst_parser",
]

autosummary_generate = True
autosummary_ignore_module_all = True

autodoc_default_options = {
  "members": True,
  "ignore-module-all": True,
  "show-inheritance": True,
}

# Keep this empty unless docs_check fails due to missing optional imports.
autodoc_mock_imports = []

html_theme = "furo"
```

- Version is read from installed package metadata, with fallback to `pyproject.toml`
- Use a fallback that works on Python 3.10+ (avoid relying on `tomllib` unless `requires-python >=3.11`)
- `autosummary_generate = True` auto-creates stub pages under `api/generated/`
- `autosummary_ignore_module_all` plus `ignore-module-all` helps avoid duplicate symbol warnings when package roots re-export classes

## autodoc_mock_imports

Only add entries if docs build actually fails due to missing optional imports.

Preferred setup in CI: install all extras for docs jobs and keep this list empty.

When you do need mocks, only use top-level import names (not deep module paths), and keep the list as short as possible.

```python
autodoc_mock_imports = []
```

## api/index.rst

```rst
API Reference

.. autosummary::
   :toctree: generated

   <package>
```

## Makefile Targets

```makefile
docs:           uv run sphinx-build -b html docs docs/_build/html
docs_check:     uv run sphinx-build -W --keep-going -b html docs docs/_build/html
docs_linkcheck: uv run sphinx-build -b linkcheck docs docs/_build/linkcheck
docs_clean:     rm -rf docs/_build docs/api/generated
```

Rules:
- `make docs` is the only command needed to build the full HTML docs
- `make docs_check` is the CI gate (warnings treated as errors)
- `make docs_clean` removes all generated output - safe to run before a fresh build
- Generated files in `docs/api/generated/` should be gitignored or treated as build output
